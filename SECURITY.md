# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue related to SWYPE, please report it privately to:

- `security@swypeswap.com`

Please do not post security issues publicly.

When possible, include:

- a short summary of the issue
- affected URL, surface, or feature
- reproduction steps
- impact assessment
- screenshots or logs if relevant

## Scope

This public GitHub presence is for official documentation and organization identity.

SWYPE proprietary application code is not published in these public repositories. Security reports may still relate to:

- the public website
- public APIs or embeds
- brand impersonation
- phishing or domain abuse
- Telegram Mini App surfaces
- account or payment-related issues

## Response

SWYPE will review submitted reports and respond as appropriate. Please avoid destructive testing, privacy violations, spam, or social engineering.
